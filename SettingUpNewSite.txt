To set up on existing managed server, add as a subdomain YOURHOST.precisionnutrition.com, e.g., dev.precisionnutrition.com

	open plesk
	Click domains
	Click precisionnutrition.com
	Click subdomains
	Click Create Subdomain
	Provide subdomain info; create subdomain name pn_YOURHOST, use existing FTP, no SSL, PHP support as apache module, click Finish
	

DocumentRoot directory is now in:
	/var/www/vhosts/precisionnutrition.com/subdomains/YOURHOST/httpdocs/
	
Add this to your LOCAL /etc/hosts file on every computer you will use to access the server by name:
	67.192.24.140 YOURHOST.precisionnutrition.com
	
Log into database server
	mysql -u root -p and enter root password
	
	CREATE DATABASE pn_YOURHOST;
	GRANT ALL PRIVILEGES ON pn_YOURHOST.* to YOURDATABASEUSER@'%' identified by 'YOURDATABASEPASSWORD';
	exit;
	
	Import the prod database dump (from db-backup) you want to use into the pn_YOURHOST database you just created
	mysql -u YOURDATABASEUSER -p pn_YOURHOST < PROD_DATABASE_DUMP_FILE.sql
	
Log into web server

	Copy all files from prod directory to the new directory. You can tar it up and copy it to a new server, but on the same server, you can:
		sudo cp -arv /var/www/vhosts/precisionnutrition.com/httpdocs/* /var/www/vhosts/precisionnutrition.com/subdomains/YOURHOST/httpdocs/
		
	Edit /var/www/vhosts/precisionnutrition.com/subdomains/YOURHOST/httpdocs/wordpress/wp-content/wp-config.php and change the database
	login information you specified above in the GRANT ALL PRIVILEGES
	
	Edit /var/www/vhosts/precisionnutrition.com/subdomains/YOURHOST/httpdocs/members/includes/config.php and change the database
	login information you specified above in the GRANT ALL PRIVILEGES. Remove any references to XCache and just use the file store
	
Copy the .htaccess in the httpdocs directory and do a search and replace on www.precisionnutrition.com with YOURHOST.precisionnutrition.com

Browse to new site in your local browser and see if it works
